# README

## ระบบจัดการธนาคารขยะในชุมชน

โปรเจกต์นี้เป็นระบบสำหรับช่วยบริหารจัดการธนาคารขยะในชุมชน โดยเน้นการบันทึกข้อมูลสมาชิก ติดตามปริมาณขยะสะสม และคำนวณเงินให้กับสมาชิกโดยอัตโนมัติ

## วัตถุประสงค์ของโปรเจกต์
- จัดการข้อมูลสมาชิกของธนาคารขยะ
- ติดตามปริมาณขยะที่สะสมของแต่ละคน
- คำนวณค่าตอบแทนจากยอดขยะอัตโนมัติ
- ให้ผู้ใช้สามารถเข้าถึงข้อมูลและติดตามสถานะได้ง่ายขึ้น

## โครงสร้างโปรเจกต์
```text
Lab1/
├── frontend/                 # แอปพลิเคชัน Frontend ที่พัฒนาโดย Next.js
│   ├── app/                  # Next.js App Router
│   │   ├── api/              # API Routes ของ Next.js (ถ้ามี)
│   │   ├── components/       # Component เฉพาะหน้า
│   │   ├── globals.css       # ไฟล์สไตล์รวม
│   │   ├── layout.tsx        # Layout หลักของแอป
│   │   └── page.tsx          # หน้าแรกของระบบ
│   ├── components/           # Component ที่ใช้ร่วมกันหลายหน้า
│   ├── lib/                  # Utility และ helper
│   ├── public/               # ไฟล์สติก เช่น รูปภาพและสื่อ
│   ├── package.json          # Dependency และสคริปต์ของ Next.js
│   ├── tsconfig.json        # ตั้งค่าคอมไพเลอร์ TypeScript
│   └── next.config.js       # การตั้งค่า Next.js
├── backend/                  # Backend/API ที่แยกออกจาก Frontend
│   ├── src/                  # โค้ดหลักของ backend
│   │   ├── controllers/      # Logic การรับ-ส่งข้อมูล
│   │   ├── routes/           # Routing ของ API
│   │   ├── services/         # Business logic
│   │   └── app.ts            # Entry point ของ API
│   ├── package.json          # Dependency และสคริปต์ของ backend
│   └── tsconfig.json        # ตั้งค่าคอมไพเลอร์ TypeScript
├── architecture.md           # เอกสารสถาปัตยกรรมระบบ
├── claude.md                 # คำแนะนำและแนวทางสำหรับการพัฒนา
├── Readme.md                 # เอกสารแนะนำโปรเจกต์
└── SRS.md                    # เอกสารข้อกำหนดซอฟต์แวร์
```

## รายละเอียดไฟล์และโฟลเดอร์
- architecture.md: บรรยายถึงสถาปัตยกรรมและโครงสร้างระบบของโปรเจกต์
- SRS.md: ระบุข้อกำหนดฟังก์ชันและคุณสมบัติของระบบ
- claude.md: แจกแจงแนวทางและคำแนะนำสำหรับการพัฒนาเพิ่มเติม
- frontend/: แอปพลิเคชันส่วนติดต่อผู้ใช้ที่พัฒนาโดย Next.js และแยกออกจาก backend
- backend/: API และบริการหลังบ้านที่รับผิดชอบการประมวลผลข้อมูลและเชื่อมต่อฐานข้อมูล
- Readme.md: เอกสารแนะนำสำหรับผู้ที่ต้องการรู้ภาพรวมของโปรเจกต์

## สถานะโปรเจกต์
โปรเจกต์อยู่ในช่วงเริ่มต้นและมุ่งเน้นการวางแผนระบบและจัดเตรียมเอกสารประกอบก่อนการพัฒนาแอปพลิเคชันจริง

## คำแนะนำสำหรับการใช้งาน
1. อ่านเอกสาร SRS.md ก่อนเพื่อเข้าใจความต้องการของระบบ
2. ศึกษา architecture.md เพื่อดูโครงสร้างและแนวทางการออกแบบ
3. ใช้ claude.md เป็นแนวทางในการพัฒนาและปรับปรุงโปรเจกต์ต่อไป
4. พัฒนาระบบโดยเริ่มจากโครงสร้าง Next.js ก่อน แล้วจึงต่อฟีเจอร์หน้าเว็บและ API routes ตามลำดับ
