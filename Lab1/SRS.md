# Software Requirements Specification (SRS)
# ระบบแจ้งเหตุเพลิงไหม้

## 1. บทนำ
ระบบแจ้งเหตุเพลิงไหม้เป็นระบบต้นแบบที่ใช้สำหรับตรวจจับควันและเปลวไฟ และแจ้งเตือนผู้เกี่ยวข้องอย่างทันท่วงที โดยระบบถูกออกแบบให้แยกส่วนการแสดงผลหน้าเว็บ (frontend) และส่วนบริการหลังบ้าน (backend/API) เพื่อให้การพัฒนาและการขยายระบบทำได้ง่ายขึ้น

## 2. วัตถุประสงค์ของระบบ
1. ตรวจจับสัญญาณของควันหรือเปลวไฟได้อย่างรวดเร็ว
2. ส่งคำเตือนให้ผู้ดูแลหรือผู้ใช้ทราบทันทีเมื่อเกิดเหตุการณ์
3. แสดงสถานะการตรวจจับและประวัติการแจ้งเตือนให้ผู้ใช้เห็นได้ชัดเจน
4. รองรับการใช้งานบนอุปกรณ์ต่าง ๆ เช่น มือถือและเดสก์ท็อป

## 3. ขอบเขตของระบบ
### 3.1 ขอบเขตในระบบ
- ตรวจสอบและแสดงสถานะการตรวจจับควัน/เปลวไฟ
- แสดงข้อความหรือการแจ้งเตือนเมื่อเกิดเหตุความเสี่ยง
- บันทึกข้อมูลเหตุการณ์และประวัติการแจ้งเตือน
- รองรับการใช้งานผ่านเว็บอินเตอร์เฟซ

### 3.2 ขอบเขตนอกระบบ
- การควบคุมอุปกรณ์ดับเพลิงหรือระบบอัตโนมัติภายนอก
- การเชื่อมต่อกับเซ็นเซอร์จริงโดยตรงในช่วงพัฒนาต้นแบบ

## 4. ผู้ใช้งานและบทบาท
### 4.1 ผู้ดูแลระบบ
- ดูสถานะระบบ
- ดูข้อมูลเหตุการณ์และประวัติการแจ้งเตือน
- กำหนดค่าระบบเบื้องต้น

### 4.2 ผู้ใช้ทั่วไป
- ดูสถานะการแจ้งเตือนของระบบ
- รับข้อความเตือนจากระบบ

## 5. รายละเอียดความต้องการระบบ
### 5.1 ความต้องการเชิงฟังก์ชัน (Functional Requirements)
- FR1: ระบบต้องสามารถแสดงสถานะการตรวจจับควันและเปลวไฟได้อย่างชัดเจน
- FR2: ระบบต้องแจ้งเตือนเมื่อตรวจพบควันหรือเปลวไฟ
- FR3: ระบบต้องแสดงข้อความเตือนในรูปแบบที่เข้าใจง่าย
- FR4: ระบบต้องสามารถแสดงประวัติการแจ้งเตือนย้อนหลังได้
- FR5: ระบบต้องรองรับการเข้าถึงผ่านเว็บบนมือถือและเดสก์ท็อป
- FR6: ระบบต้องมีส่วนจัดการข้อมูลพื้นฐานของเหตุการณ์และสถานะต่าง ๆ

### 5.2 ความต้องการเชิงไม่ใช่ฟังก์ชัน (Non-Functional Requirements)
- NFR1: ระบบต้องตอบสนองต่อคำขอได้เร็วภายใน 3 วินาที
- NFR2: ระบบต้องมีความเสถียรและพร้อมให้บริการอย่างต่อเนื่อง
- NFR3: ระบบต้องมีความปลอดภัยในการเข้าถึงข้อมูล
- NFR4: ระบบต้องมีอินเตอร์เฟซที่ใช้งานง่ายและเข้าใจง่าย
- NFR5: ระบบต้องสามารถพัฒนาและต่อยอดได้ในอนาคต

## 6. Use Case
### UC1: ดูสถานะระบบ
ผู้ใช้เข้าสู่ระบบและดูสถานะปัจจุบันของการตรวจจับควันและเปลวไฟ

### UC2: ตรวจพบเหตุการณ์อันตราย
ระบบตรวจพบสัญญาณควันหรือเปลวไฟและสร้างการแจ้งเตือน

### UC3: ดูประวัติการแจ้งเตือน
ผู้ดูแลสามารถดูข้อมูลเหตุการณ์ที่เกิดขึ้นในอดีต

## 7. ข้อกำหนดด้านข้อมูล
- ระบบต้องเก็บข้อมูลสถานะล่าสุดของการตรวจจับ
- ระบบต้องเก็บประวัติการแจ้งเตือนพร้อมวันเวลาที่เกิดเหตุ
- ข้อมูลที่เก็บต้องสามารถเรียกดูเพื่อใช้งานต่อในอนาคต

## 8. สมมติฐานและข้อจำกัด
- ระบบนี้เป็นต้นแบบ ดังนั้นฟังก์ชันการตรวจจับจะเน้นที่การแสดงผลและการแจ้งเตือนแบบพื้นฐาน
- การเชื่อมต่อกับเซ็นเซอร์จริงหรืออุปกรณ์ภายนอกอาจต้องพัฒนาเพิ่มเติมในอนาคต
- โครงสร้างระบบจะใช้ Next.js สำหรับส่วนหน้าเว็บและ Backend/API สำหรับส่วนบริการหลังบ้าน

## 9. เกณฑ์การยอมรับ
ระบบถือว่าผ่านเกณฑ์เมื่อ
1. สามารถแสดงสถานะการตรวจจับควันและเปลวไฟได้
2. สามารถส่งการแจ้งเตือนเมื่อเกิดเหตุการณ์ได้
3. สามารถแสดงประวัติการแจ้งเตือนได้อย่างถูกต้อง
4. ใช้งานได้ทั้งบนมือถือและเดสก์ท็อป
